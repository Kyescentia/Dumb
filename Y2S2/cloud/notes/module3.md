# **Module 3: AWS Global Infrastructure Overview**

#### **Section 1: AWS Global Infrastructure**
* **Regions:** Geographical areas containing multiple Availability Zones (AZs). Selection is based on data governance, latency, service availability, and cost.
* **Availability Zones (AZs):** Fully isolated partitions within a Region. Each consists of discrete data centers with redundant power, networking, and connectivity. They are interconnected via high-speed private networking.
* **Data Centers:** Secure facilities housing thousands of servers.
* **Edge Locations:** Points of Presence used by Amazon CloudFront to cache content closer to users for lower latency.

#### **Section 2: AWS Services and Service Category Overview**
* **Foundational Services:**
    * **Compute:** EC2, Lambda, ECS, EKS.
    * **Storage:** S3, EBS, EFS, Glacier.
    * **Networking:** VPC, Route 53, Direct Connect, Elastic Load Balancing.
* **Platform Services:** Databases (RDS, DynamoDB), Analytics, Mobile, IoT, Developer Tools, etc.
* **Service Categories:** Include Security (IAM, KMS), Management (CloudWatch, CloudTrail, Trusted Advisor), and Cost Management (Budgets, Cost Explorer).
